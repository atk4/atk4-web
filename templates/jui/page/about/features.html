Agile Toolkit features

Agile Toolkit is a PHP5 framework developed by Agile Technologies. The Toolkit includes a solid MVC framework, a large number of ready-to-use widgets and controllers and it is specifically designed for lean, agile Web Software development.

Agile Toolkit is split in 2 components:

  • Agile Toolkit Core - this provides efficient integration to any other frameworks.
  • ATK Interface Library - introduces set of visual components and speeds up development of UI incredibly.

Features Benefits
1. Highly re-usable modules A typical Software as a Service (SaaS) application re-uses 80% of code written for other similar applications. The Agile Toolkit has developed this common code as modules that you can use so that you can focus your effort on the 20% that makes your code different.

Re-usage modules mean:
  1. Takes less time to build your application
  2. Robust, scalable code build for expanding on
  3. Enables you to focus on the unique parts of your software
2. Front end interface design About 30% of the time and budget in each project is spend developing the front end HTML & User Interface (UI). User Interface is often the most powerful selling point for SaaS applications so here's what we do t make it easier.

Agile Tookit UI advantages:
  1. It allows the interface throughout the application
  2. The Developer can create simple user interface without involving a designer or knowing HTML
  3. The Designer can develop html templates independently of development. Later either the developer or designer can apply them to the software with ease.
  4. The Agile Toolkit UI follows design guidelines and web standards
3. Security is built-in The Agile Toolkit is built according to general practices and recommendations for secure software published by OWASP. This takes the responsibility of maintaining security away from developers.

What it means for the developer:
  1. You don't have to worry about security vulnerabilities
  2. There are Less testing and audits involved
  3. Security standards are already in place
4. Business goal oriented design Agile Toolkit was built with a business owners in mind. The framework requires less training and has extensive documentation for every business feature you would need.

  1. Less training involved
  2. Business goals can be achieved in minutes
  3. Simple and extensive documentation
5. Newest technology Microsoft, Apple and Google are collaboratively promoting newest technological standards for web. For other companies to take advantage of those new technologies takes a lot of learning and getting used to.

Agile Framework allows you to:
  1. Produce great results with HTML5, JavaScript and CSS
  2. Includes technologies such as jQuery, jQuery UI and MVC
  3. Creates richer client-side software using AJAX technologies
6. Less dependancies between team members A typical web project includes an architect, user interface designer, developer, and business analyst. Usually each of these depends on deliverables from the other in order to carry out their work. What if more of their work could be carried out independently of each-other?

Th Agile Toolkit is designed for:
  1. Less management and paperwork
  2. Each person can implement what they know best.
  3. This is easier to merge than with other frameworks
  4. You don't have to put them in one room to deliver result
7. Fast and highly scalable The Toolkit use Object-Oriented-Design with each components separated in Modules. For example, in a social networking application, when a logged in user looks at their profile page, the profile page does not need the code responsible for "login". Instead this is in the login module with only a tag necessary on the profile page.

In other words, visual elements such as Forms, Grids, Tables and Menus are dynamically loaded only as required. If AJAX is used to re-draw a Form then the other components on the page are excluded from the form. A similar approach works on front end with JavaScript, jQuery and jQuery UI components loaded on demand. This makes software fast and efficient.
8. Integrated Many frameworks decide to have as few dependencies on other components as possible. Using them often limits the developer in terms of licensing and leaves a huge code footprint.
In contrast, the Agile Toolkit relies on standard components such as XML, DOM and XSLT to perform all the tasks. Most of the work is done by relevant PHP extensions.

  1. This makes your solution highly customisable and extensive, Easier to maintain in long term, but we also esure that there is no licensing implications on 3rd party modules.
