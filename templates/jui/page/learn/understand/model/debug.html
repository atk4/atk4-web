Debugging Models

Important thing about models you should understand, is that models on their own are here to serve 3 purposes:

  • Produce dynamic SQL objects
  • Load, hold and save one record of data
  • Contain meta-information for other views
  • Perform business operations
Some actions are more likely to go wrong, usually because of incorrectly defined fields.

Debugging produced SQL statement

You can switch on debug mode for any model by calling: $model->debug();

This will output queries as they are being generated by the underlying Dynamic SQL object(s). This method is good to see how dynamic SQL queries are produced or how data is being loaded, saved or otherwise accessed.

Get all data from model

You can see what's inside your model by calling var_dump($model->get()); This will output all the data saved in the model.

Transaction support

By default operations of a model are atomic and it does not use SQL transactions. However if you are using more complex operations, it's advised to use transaction support:

class Model_Book extends Model_Table { // Skipped function addMultipleBooks($data){ $this->api->db->beginTransaction(); $this->unloadData(); foreach($data as $row){ $this->insert($data); } $this->api->db->commit(); } }

If error happens during inserts, exception will bubble up through the call-stack and data will not be committed.

Adding more information to exceptions

In Agile Toolkit error reporting is implemented using Exceptions. To improve reporting you can intercept exception add more informations to them.

function addMultipleBooks($data){ $this->api->db->beginTransaction(); $this->unloadData(); try { foreach($data as $row){ $this->insert($data); } }catch(BasicException $e){ throw $e->addMoreInfo('record_count',count(data)); } $this->api->db->commit(); }

Catching exception will allow you to manipulate with it such as calling addMoreException. This method returns "$this" of exception which you are throwing again and which will be handled by the default application error handler, but your additional information will be included.

exception() method

Agile Toolkit defines method exception() globally. Each model can define which exception class it is using and exception() function will return instance of this class. This also allows a more flexible syntax in PHP code. (throw $this->exception()->addMoreInfo(..))

You can either redefine this method to add additional information (such as certain data from model) for exceptions which are being generated or specify a different exception class through $default_exception property.
