Knowledge Requirements

Developing on Agile Toolkit is much much simpler than developing with plain PHP. It makes great sense to skip some stuff and get real results quick.

Agile Toolkit is used to power many high end projects and it does not lack in flexibility or power. Once you start using it, it's unlikely that you'll want to mess with things on the low level.

It turns out that you need to know very little to use Agile Toolkit.

Programming in PHP

It might sound sad, but PHP is not the best language to learn programming with. You can write the code but seeing result and experimenting is quite challenging. Much less straightforward than QBASIC used to be.

However you would need to know about PHP expressions, variables, control structures logical operators, so start reading language reference. You might want to skip features which Agile Toolkit specifically does not use (as noted on previous page)

Object-Oriented Programming Training

Apart from syntax, you need to understand how objects work. Look for some on-line documentation for full information, but you would need to understand the difference between Class and Object, how class inheritance works and how you can override methods. Some experience in object-oriented development is really beneficent for Agile Toolkit even if you used language other than PHP.
